[1] java의 장점

- jvm이 중간에서 java 파일을 운영체제가 이해할 수 있는 언어로 바꿔주기 때문에 os의 종류에 구애받지 않는다. (어디서든 실행이 가능해진다.)

- 메모리 관리를 할 필요가 없다. 자바의 가비지 컬렉터가 자동으로 해 주기 때문이다. 

- 멀티 쓰레드 프로그래밍의 경우 환경에 맞게 최적화해야 하는 부분이 있는데 java가 제공하는 api를 사용해서 편리하게 할 수 있다.

- 객체 지향적인 언어의 장점을 최대한 발휘할 수 있다. (상속, 캡슐화, 다형성)

- 동적 로딩을 제공한다. (모든 클래스를 로딩하지 않고 필요한 시점에 로딩 -> 유연하게 애플리케이션을 변경할 수 있음)

<br>

[2] jvm

java virtual machine
자바를 실행시켜주는 가상의 컴퓨터. jvm을 통해서 자바는 os 독립적인 언어가 될 수 있었다.

다른 언어가 어플리케이션 - os - 하드웨어만 거치는 데 비해 java는 어플리케이션 - jvm - os - 하드웨어 이렇게 하나의 계층이 더 있어 속도 측면에서 불리했었다. (현재는 JIT컴파일러와 최적화 기술이 발달해서 어느 정도 극복함)

<br>

[3] 자바 어플리케이션이 실행되는 과정

javac.exe가 .java 파일을 컴파일해서 .class 파일로 만든다.

java.exe(인터프리터)가 .class 파일을 실행한다.

프로그램에 필요한 클래스 파일을 로드하면서 클래스 파일을 검사한다. (악성코드, 파일 형식)

지정된 클래스 내에서 main을 호출한다.


<br>

[4] main 메서드

자바는 하나의 클래스에 여러 개의 하위 클래스를 가질 수 있다.

하나의 클래스에 main 메서드는 하나는 꼭 있어야 한다.

자바 소스코드 명은 public이 붙은 클래스가 있으면 해당 클래스의 이름을 따라야 하고, 없는 경우 어떤 클래스의 이름으로 해도 상관은 없다.
