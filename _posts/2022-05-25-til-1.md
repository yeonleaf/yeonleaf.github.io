연산 과정에서 산술변환이 자동으로 일어난다. (작은 타입을 큰 타입으로 일치시켜서 계산, int보다 작은 값은 int로 일치시키기)

```java
1) 10.0==10.0f
2) 0.1==0.1f
```
1) double을 float으로 변환하더라도 float이 충분히 담을 수 있는 값이기 때문에 오차가 적다. -> True
2) double을 float으로 변환하는 과정에서 오차가 발생 -> false


```java
char c1 = 'a';
char c2 = c1 + 1; // 1)
char c2 = 'a' + 1; // 2)
```
똑같은 코드처럼 보이지만 1)에서는 컴파일 에러가 발생한다. 2)는 리터럴 간의 연산이므로 컴파일러에서 미리 계산을 할 수 있지만 변수명이 들어간 1)은 불가능하기 때문이다.


논리 연산자(and, or)의 특징은 효율적인 연산

or의 경우 첫 번째 구문이 true이면 두 번째 구문은 확인하지 않는다. and의 경우 첫 번째 구문이 false이면 두 번째 구문은 확인하지 않는다.


& | ^ (and or xor)
and : 두 값 모두 1일 경우 1
or : 둘 중의 하나라도 1일 경우 1
xor : 두 값이 같으면 0, 다르면 1


~ 비트 전환 연산자 (1의 보수 연산자)

어떤 수에서 부호만 바뀐 수를 얻고 싶다면

~p+1


쉬프트 연산자
```java

<< : 십진수를 이진수로 바꾼 후 왼쪽으로 두 칸 밀기 (*2와 같음)
8 << 0
8 << 1
8 << 2

>> : 십진수를 이진수로 바꾼 후 오른쪽으로 두 칸 밀기 (/2와 같음)
8 >> 0
8 >> 1
8 >> 2

옮긴 다음 빈 공간은 양수일 경우 0으로, 음수일 경우 1로 채움
```
