[1] 변수

하나의 값을 저장할 수 있는 공간
변수를 선언하면 메모리의 빈 공간에 변수 타입에 알맞은 저장공간이 확보되고, 변수 이름을 통해 해당 공간을 사용할 수 있게 된다.

```java
int age;
// 변수 타입 + 변수 명
```

초기화 : 변수를 선언한 후 처음으로 값을 저장하는 것 (선언 후 초기화를 반드시 해 줘야 한다. 다른 프로그램에서 실행하고 남은 값이 있을 수도 있기 때문에)


(1) 변수의 명명규칙

- 예약어를 사용해서는 안 된다.
- 대소문자를 구분한다.
- 숫자로 시작하면 안 된다.
- 특수문자는 $, _만을 허용

(2) 변수의 타입

- 기본 타입

논리형, 문자형, 정수형, 실수형
boolean, char, byte, short, int, long, float, double

8bit == 1byte
```
1byte boolean, byte
2byte char, short
4byte int, float
8byte long, double
```

각각의 타입은 -2^n ~ 2^n (n은 bit) 까지의 값을 저장할 수 있다.

정밀도는 오차없이 저장할 수 있는 범위를 의미한다.

float의 정밀도는 7자리, double의 정밀도는 15자리이다. (double 쓰시오)


- 참조 타입

기본 타입을 제외한 모든 것

(3) 상수와 리터럴

상수는 초기화 이후 값을 변경할 수 없는 공간을 의미한다. 한 번 저장하면 값을 바꿀 수 없는 공간

상수를 저렇게 정의하면서 기존에 상수라는 의미를 가지고 쓰여지던 10, 'A' 이런 값들을 어떻게 불러야 할지 난감해졌는데 여기다 리터럴이라는 이름을 붙이게 되었다.

```java
final int A;
```

이렇게 앞에다 final을 붙여 주면 상수 선언을 할 수 있다.

일반적으로 상수는 대문자로 선언한다. (just 관행)


리터럴 중에서는 특별한 의미를 가지는 리터럴이 있다.

접미사가 l, L이면 long, f, F이면 float, d, D이면 double이다.

int, double은 각각 4바이트, 8바이트 자료형의 대표이므로 리터럴을 붙이지 않아도 된다.

따라서 실수형인데 리터럴이 생략되어 있으면 double이라고 보면 된다.

```java
float a = 3.14;
```

따라서 이런 대입은 안 된다. 3.14는 리터럴이 생략되어 있기에 double인데 double은 그보다 작은 float에 들어갈 수 없기 때문이다.


'A'는 문자 리터럴. 두 문자 이상은 큰따옴표로 감싸며 문자열 리터럴이라고 부른다.

문자열 리터럴은 빈 문자 ""을 허용하지만 문자 리터럴은 허용하지 않는다. 


문자열 + 다른 type 이면 다른 타입을 먼저 문자열로 형변환 한 후 덧셈 연산을 수행한다.

[5] 진법

n비트로 표현할 수 있는 값은 2^n

10진법 소수를 2진법 소수로 바꾸기
1. 10진법 소수에 2를 곱한다.
2. 소수점 아래의 값만 남긴다.
3. 1과 2를 반복한다. (소수부가 0이 될 때까지)
4. 정수부만 적으면 2진법 소수가 된다.

2진법 소수를 10진법 소수로 바꾸기
0.101
1*(2^-1) + 0*(2^-2) + 1*(2^-3)

음수를 이진수로 바꾸기 (2의 보수법)

1. 음수의 절대값을 이진수로 바꾼다.
2. 1의 1의 보수 (1은 0으로, 0은 1로 바꿈)를 구한다.
3. 2에 1을 더한다.
