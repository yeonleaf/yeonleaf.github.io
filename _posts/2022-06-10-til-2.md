[1] 클래스 / 인스턴스 메서드

클래스 변수 / 클래스 메서드는 인스턴스 메서드를 호출할 수 없다. 클래스가 생성된 시점에 인스턴스가 존재하지 않을 수도 있기 때문이다.

<br>

[2] 오버로딩

오버로딩은 하나의 클래스에 같은 이름의 메서드를 여러 개 정의하는 것이다.
매개변수의 타입, 개수, 순서가 달라야 한다. 리턴타입은 오버로딩에 영향을 미치지 않는다.
순서를 기억해 두자.
```java
public Long add(Long a, int b);
public Long add(int a, Long b);
```
이 경우 오버로딩으로 인정이 되는 이유는 매개변수를 넣는 시점에서 구별이 되기 때문이라고 한다.


가변변수 (args)의 경우 내부적으로 배열을 생성하는 식으로 동작하는데 (이 때문에 가변변수를 사용하기 전에 이게 정말 필요한지 생각은 해 볼 필요가 있다.) 
매개변수를 하나도 넣지 않아도 된다는 장점이 있다. 배열을 매개변수 타입으로 지정하면 길이가 0인 배열이나 null값을 반드시 세팅해 주어야 한다.

<br>

[3] 생성자

```java
new Member();
```
여기서 새로운 객체를 생성하는 것은 new 키워드이고 Member()는 기본 생성자를 호출한다.

이처럼 생성자는 새로운 객체를 생성하는 것이 아니라 인스턴스 생성 시에 변수를 초기화하는 데 도움을 주는 특별한 메서드라고 생각을 하면 된다.

생성자가 하나도 없는 경우에는 java가 기본 생성자 하나를 생성해준다. 반대로 생각하면 다른 생성자가 있는 경우에는 java가 기본 생성자를 만들어 주지 않기 때문에 인스턴스를 생성하는 과정에서 에러가 발생할 수 있다.

생성자에서 다른 생성자를 호출하기 위해서는 this()를 사용한다. 

```java
Car() {
    this("white", "auto", 4); // 다른 생성자를 호출
}
```

this는 인스턴스 자기 자신을 가키는 참조변수이기도 하다. 여기에는 인스턴스의 주소가 저장되어 있다.
매개변수와 지역변수가 같은 경우가 종종 생기기 때문에 생성자에서 자기 자신 인스턴스를 지정하고 싶다면 this를 사용하는 것이 좋다.

```java
public Member(String name, int age) {
    name = name;
    age = age;
}
```
name = name에서 앞에 있는 name은 지역 변수이다.

```java
public Member(String name, int age) {
    this.name = name;
    this.age = age;
}
```
이렇게 하면 인스턴스 변수를 가리킬 수 있다.

<br>

[4] 변수의 초기화

멤버변수는 초기화를 하지 않아도 기본 자료형으로 초기화가 되지만 지역 변수는 사용하기 전에 반드시 초기화를 해야 한다.

변수 초기화의 방법에는 아래의 세 가지가 있다.

- 명시적 초기화
```java
int iv = 0;
```

- 생성자
```java
Obj(int iv) {
    this.iv = iv;
}
```
- 초기화 블럭
    - 클래스 초기화 블럭 : 클래스가 메모리에 처음 로딩될 때 한 번만 수행됨
    - 인스턴스 초기화 블럭 : 인스턴스가 생성될 때마다 호출된다.
    ```java
    static int count;
    {
        count ++;
    }
    ```
    이런 식으로 코드를 짜면 인스턴스가 생성될 때마다 초기화 블럭 내부의 count++이 클래스 멤버변수의 값을 변경하게 된다.

    - 클래스 변수의 초기화 과정 : 기본값 -> 명시적 초기화 -> 클래스 초기화 블럭
    - 인스턴스 변수의 초기화 과정 : 기본값 -> 명시적 초기화 -> 인스턴스 초기화 블럭 -> 생성자
