[1]

네트워킹 : 두 대 이상의 컴퓨터를 케이블로 연결하여 서로 네트워크를 주고받는 것

서비스 : 클라이언트에게서 요청 받은 작업을 받아서 결과를 반환하는 과정

클라이언트 : 서비스를 이용하는 컴퓨터

서버 : 서비스를 제공하는 컴퓨터

가정용 PC는 대부분 클라이언트 역할을 담당하지만 tomcat 같은 웹 서버를 설치한다면 서버의 역할도 할 수 있다.

서버 기반 모델 : 전용 서버 컴퓨터가 있는 겻

- 안정적인 서비스가 가능
- 서버 구축 비용, 관리 비용이 듬

peer to peer방식 : 프로그램에 클라이언트 + 서버를 통합해서 컴퓨터가 클라이언트가 됨과 동시에 서버의 역할도 수행하게 하는 것

- 서버 구축 및 운용비용을 절감할 수 있다.
- 보안 문제가 있음


[2]

IP : 컴퓨터마다 주소를 가지고 있음 32bit

IP주소는 네트워크 주소 + 호스트 주소로 나뉨

어디까지가 네트워크 주소이고 호스트 주소인지는 네트워크의 구성 방식에 따라서 달라짐

그렇다면 네트워크 주소를 IP주소에서 어떻게 분리?

네트워크 주소를 구하고 싶은 IP 주소와 서브넷 마스크(255.255.255.0)를 &연산하면 된다.

IP주소를 다루기 위한 클래스는 InetAddress


[3] 

URL (Uniform Resource Locator)
자원에 접근하는 주소

프로토콜 : 도메인 : 포트 번호 : 경로 : 파일명 : 쿼리 : 참조(anchor : # 이후의 부분)

URL을 다루기 위한 클래스는 URL

openConnection() - URL과 연결된 URLConnection을 가져온다.

[4]

URLConnection : URL과 어플리케이션의 연결을 담당하는 객체. 이 객체를 통해서 자원에 접근해서 읽고 쓰기를 할 수 있다.
추상 클래스이며 구현체로 HttpURLConnection과 JarURLConnection이 있다. 어떤 프로토콜로 들어오는지에 따라 서로 다른 구현체를 반환한다.

openStream() -> openConnection() + getInputStream() -> inputStream
