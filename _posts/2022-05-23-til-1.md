[1] 정수

byte < short < int < long

이 중 기본은 int

1byte -> 2byte -> 4byte -> 8byte

bit로 변환 : byte에 * 8

n = bit

표현 범위 -2^n-1 ~ 2^n-1-1

-1을 빼는 이유는 0 때문

정수는
(부호) (타입의 크기)
로 이루어진다.

byte와 short가 용량이 작기는 하지만 jvm의 operand stack이 byte와 short를 4바이트로 저장하므로 결과적으로는 int를 사용하는 것이 좋다.

20억이 넘어가는 정수의 경우 long을 사용하면 되고, long의 범위 (-2^63~2^63-1)을 넘어가는 경우 BigInteger 클래스를 사용한다.

정수형에서는 overflow가 발생하는데, 최소값에서 -1하면 최대값, 최대값에서 +1하면 최소값이 된다. (타입 선택이 중요한 이유)
이렇게 되는 이유는 최대값에서 +1하면 올림이 발생하는데 저장할 공간이 없어서 버려지기 때문이다. 최소값의 경우도 마찬가지

부호가 없는 정수의 경우 1111 -> 0000으로 넘어가는 순간 오버플로우가 발생하고 (0~15가 반복됨) 부호가 있는 정수의 경우 7에서 -8로 넘어갈 때 (0111 -> 1000) 오버플로우가 발생한다.

short와 char는 둘 다 2byte이기 때문에 범위가 같을 것 같지만 short는 음수도 표현하는 반면 char는 양수만 표현하기 때문에 실제로는 범위가 다르다. (형변환이 안 되는 이유)

[2] 실수

저장 방식 자체가 정수와 다르다.

정수는 음수로 바꿀 때 2의 보수법을 사용하는 반면 실수는 2의 보수법을 사용하지 않고 기저법을 사용한다.

float -> double
4byte -> 8byte

(부호)(지수)(가수) 형태로 이루어져 있다.

float  : 1 + 8 + 23 (32 bit)

double : 1 : 11 : 52 (64 bit)

이런 구조 때문에 float와 int는 둘 다 4byte이지만 float은 지수가 곱해져 있는 형태이기 때문에 실제로는 저장공간이 더 넉넉하다고 봐야 한다.

2진법 실수를 저장할 때는 정규화한 후 자료형에 따라 float은 가수를 23bit, double은 52bit까지만 저장한다. 이 차이가 float과 double의 정밀도 차이가 된다. 2^23은 십진수로 바꾸면 10^7, 2^52는 10^15 정도가 되기 때문이다.

이 정밀도 차이 때문에 만약에 float의 경우 정규화한 실수의 24번째 자리가 1, double의 경우 53번째 자리가 1이면 반올림이 발생해서 원래의 값과 달라질 수 있다.

```java
Float.floatToIntBits();
```
이 메서드는 float타입을 int타입으로 해석해서 반환한다.

[3] 형변환

기본형 -> 기본형 간의 형변환은 boolean을 제외하고 허용된다.

기본형 -> 참조형 간의 형변환은 불가능하다.

작은 것 -> 큰 것의 형변환은 (타입)을 붙여주지 않아도 허용된다. (값의 손실이 없기 때문)


- 정수 -> 정수 간의 형변환

    큰 것 -> 작은 것으로 형변환하면 경우에 따라 값의 손실이 발생할 수 있다. 혹시 잘려나가는 bit 옆 bit가 1이면 반올림이 적용되어 오차가 발생할 수도 있다.

    작은 것 -> 큰 것으로 형변환할 경우 남는 자리를 양수는 0, 음수는 1로 채운다. (2의 보수법)

- 실수1 -> 실수2 간의 형변환

    지수 : 실수1의 기저를 뺀 후 실수2의 기저를 더한다.

    가수 : 아래 참고
        
    - 실수 1 < 실수 2인 경우 (작은 타입에서 큰 타입으로)

        남는 자리를 0으로 채운다.
        
    - 실수 1 > 실수 2인 경우
        
        실수 2의 가수 자리만큼만 실수 1을 채워넣고 나머지는 반올림한다.
